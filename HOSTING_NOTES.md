## Fastest way to host again

### Option A: GitHub Pages

1. Create a new GitHub repository in an account you control.
2. Copy these files into it:
   - `index.html`
   - `CNAME` if you will use `www.strandrayan.bar`
3. In GitHub, go to repository `Settings > Pages`.
4. Set source to `Deploy from a branch`, branch `main`, folder `/root`.
5. If using a custom domain, set it to `www.strandrayan.bar`.
6. In GoDaddy DNS, update:
   - `www` CNAME -> `<your-github-username>.github.io`
   - apex/root `@` A records -> GitHub Pages IPs:
     - `185.199.108.153`
     - `185.199.109.153`
     - `185.199.110.153`
     - `185.199.111.153`

Alternative for apex/root: keep the website on `www` and add a GoDaddy forwarding rule from `strandrayan.bar` to `https://www.strandrayan.bar/`.

### Option B: Netlify

1. Create a Netlify account.
2. Add a new site by drag-and-dropping this folder, or connect your GitHub repo.
3. Build command: none.
4. Publish directory: `/` or repository root.
5. Add the custom domain in Netlify.
6. Update GoDaddy DNS to the records Netlify provides.

### Option C: Vercel

1. Create a Vercel account.
2. Import the repo.
3. Framework preset: Other/static.
4. Build command: none.
5. Output directory: leave empty/root.
6. Add the custom domain and update GoDaddy DNS as Vercel instructs.
